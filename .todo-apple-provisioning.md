# TODO: Apple Developer Portal — Provisioning Profile for WebAuthn

## Context

`ASAuthorizationController` requires Associated Domains, which requires the restricted
`com.apple.developer.web-browser.public-key-credential` entitlement, which requires a
provisioning profile. This is a self-service process — no special Apple review needed.

## Steps

1. Log in to https://developer.apple.com/account
2. **Identifiers** → `+` → App IDs → register `com.cmuxterm.app.lab`
3. On that App ID → Additional Capabilities → enable **"Web Browser — Public Key Credential"**
   (if it requires a request form: short form, "we're building a browser", ~1-2 day approval)
4. **Profiles** → `+` → Developer ID → select the App ID → select cert → download `.provisioningprofile`
5. In fork-release.yml, embed profile at `Contents/embedded.provisioningprofile` before codesigning
6. Re-add `com.apple.developer.web-browser.public-key-credential` to `cmux.entitlements`
7. Tag `lab-v0.64.0` → test

## CI workflow change needed

After the build step, before codesign:
```bash
# Embed provisioning profile
cp cmux-lab.provisioningprofile "$APP_PATH/Contents/embedded.provisioningprofile"
```

The profile would need to be base64-encoded as a GitHub secret (like the cert).
